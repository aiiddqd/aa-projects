# SAFETY.MD

Универсальная политика безопасности, загружается в каждой агентной сессии.

## 1. Уровни Кризисности
```yaml
severity_levels:
  s0: "Нет риска: штатная работа"
  s1: "Незначительное несоответствие: низкое влияние на аудиторию"
  s2: "Существенный риск: некорректные заявления или комплаенс-риск"
  s3: "Критический инцидент: юридический, репутационный или бюджетный риск"
```

## 2. Kill Switch
```yaml
kill_switch:
  owner: "human_marketing_owner"
  trigger_conditions:
    - "опубликовано неподтвержденное публичное утверждение"
    - "обнаружено нарушение suppression-правил"
    - "аномалия расходов выше согласованного порога"
    - "активная юридическая или комплаенс-претензия"
  actions:
    - "pause_all_outbound_sends"
    - "pause_all_paid_campaigns"
    - "lock_autonomous_publishing"
```

## 3. Матрица Эскалации
```yaml
escalation:
  s1: ["marketing_owner"]
  s2: ["marketing_owner", "compliance_owner"]
  s3: ["marketing_owner", "compliance_owner", "founder_or_exec"]
response_sla:
  s1: "в течение 1 рабочего дня"
  s2: "в течение 4 часов"
  s3: "немедленно"
```

## 4. Шаблоны Публичной Реакции
- Проверка инцидента:
  Мы проверяем информацию и временно приостановили связанные активности до завершения верификации.
- Корректирующие действия:
  Мы выявили причину, внесли исправления и включили дополнительные предохранители для предотвращения повторения.

## 5. Заблокированные Темы И Утверждения
- Юридические гарантии и комплаенс-интерпретации без согласования человеком.
- Имена клиентов, логотипы и отзывы без явного разрешения.
- Статистика эффективности без источника и владельца согласования.

## 6. Критерии Перезапуска
```yaml
restart_requirements:
  - "root_cause_documented"
  - "corrective_actions_applied"
  - "owner_approval_recorded"
  - "monitoring_checks_enabled"
```
